# Course 4: Deploying Machine Learning Models in Production

This folder contains the notebooks, Python scripts, labs, and learning materials completed as part of **Course 4: Deploying Machine Learning Models in Production**, from the **Machine Learning for Production (MLOps) Specialization**, offered by **DeepLearning.AI** through **Coursera**.

This course focuses on the final stage of the production machine learning lifecycle: making models available to users and keeping them reliable over time. It covers model serving, inference infrastructure, workflow automation, model versioning, continuous delivery, progressive delivery, monitoring, logging, and model decay.

The course also introduces important operational and responsible AI topics, including observability, privacy, secure AI systems, and legal considerations related to production ML.

## Course Overview

This course explains how machine learning models are deployed and maintained in real production environments.

The main idea is that deployment is not just about exposing a model endpoint. A production ML system also needs reliable serving infrastructure, scalable inference, model versioning, workflow automation, delivery strategies, monitoring, logging, and procedures to detect and respond to model performance degradation.

The course covers both real-time and batch inference scenarios, as well as tools and practices such as TensorFlow Serving, Docker, FastAPI, Kubernetes, Kubeflow Pipelines, TFX, GitHub Actions, CI/CD, canary releases, and model monitoring.

## Course Structure

| Week | Topic | Summary |
|---|---|---|
| Week 1 | Model Serving Introduction | Introduced the basics of making ML models available to end users. The week covered model serving concepts, serving infrastructure, deployment options, latency and cost optimization, TensorFlow Serving, Docker, and introductory cloud deployment workflows. |
| Week 2 | Model Serving Patterns and Infrastructures | Covered different ways to serve models for real-time and batch inference. The week introduced serving architectures, TensorFlow Serving, FastAPI with Docker, scalable infrastructure, Kubernetes, Kubeflow, load testing, online inference, data preprocessing, and batch processing with ETL workflows. |
| Week 3 | Model Management and Delivery | Focused on managing ML workflows in a more production-oriented way. The week covered experiment tracking, MLOps maturity levels, workflow orchestration, TFX custom components, Kubeflow Pipelines, model versioning, GitHub Actions, CI/CD, continuous delivery, progressive delivery, and canary releases. |
| Week 4 | Model Monitoring and Logging | Covered how to monitor production ML systems after deployment. The week introduced observability, logging, tracing, monitoring targets, model decay, strategies to mitigate performance degradation, responsible AI, privacy requirements, anonymization, pseudonymization, and regulations such as GDPR and CCPA. |
