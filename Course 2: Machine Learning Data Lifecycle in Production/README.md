# Course 2: Machine Learning Data Lifecycle in Production

This folder contains the notebooks, Python scripts, labs, and learning materials completed as part of **Course 2: Machine Learning Data Lifecycle in Production**, from the **Machine Learning for Production (MLOps) Specialization**, offered by **DeepLearning.AI** through **Coursera**.

This course focuses on one of the most important parts of production machine learning: the data lifecycle. It covers how to collect, clean, validate, transform, track, and manage data so that machine learning systems can work reliably over time.

While the first course introduced the general ML production lifecycle, this course goes deeper into the data layer. The main emphasis is on building production-ready data pipelines using tools such as TensorFlow Extended, TensorFlow Data Validation, TensorFlow Transform, ML Metadata, schemas, and feature engineering workflows.

## Course Overview

This course explains how data moves through a production ML system, from collection and labeling to validation, transformation, storage, and monitoring over time.

The course highlights that production ML depends heavily on data quality. A model can degrade not only because of modeling issues, but also because the data changes, labels become inconsistent, schemas evolve, or preprocessing steps are not reproducible.

The course also introduces practical techniques for improving datasets, including feature engineering, feature selection, active learning, weak supervision, semi-supervised learning, and data augmentation across different data types.

## Course Structure

| Week | Topic | Summary |
|---|---|---|
| Week 1 | Collecting, Labeling, and Validating Data | Introduced the role of data in production ML systems and why data quality is central to model performance. The week covered ML pipelines, responsible data practices, data and concept changes, human labeling, and detecting data issues using TensorFlow Data Validation. |
| Week 2 | Feature Engineering, Transformation, and Selection | Focused on transforming raw data into useful model inputs. The week covered preprocessing operations, feature engineering techniques, feature crosses, scalable transformations with TensorFlow Transform, feature spaces, and different approaches to feature selection. |
| Week 3 | Data Journey and Data Storage | Covered how data evolves across the lifecycle of a production ML system. The week introduced ML Metadata, schema development, schema environments, feature stores, data warehouses, and data lakes as tools for tracking, organizing, and managing production data. |
| Week 4 | Advanced Data Labeling Methods, Data Augmentation, and Preprocessing Different Data Types | Explored techniques for improving datasets when labeled data is limited or when different data types require specialized preprocessing. The week covered semi-supervised learning, active learning, weak supervision, data augmentation, and feature engineering for weather data, accelerometer data, time series, signals, and images. |
