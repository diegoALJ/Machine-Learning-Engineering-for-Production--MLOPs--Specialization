# Course 1: Machine Learning in Production

This folder contains the notebooks, Python scripts, labs, and learning materials completed as part of **Course 1: Machine Learning in Production**, from the **Machine Learning for Production (MLOps) Specialization**, offered by **DeepLearning.AI** through **Coursera**.

This first course introduces the main ideas behind production machine learning systems. It covers the full ML lifecycle from project scoping and data definition to modeling, deployment, monitoring, and continuous improvement.

Unlike a traditional machine learning course focused mainly on training models, this course explains what is needed to make ML systems work reliably in real-world environments, where data changes over time and models need to be monitored, maintained, and improved.

## Course Overview

This course provides a high-level framework for thinking about machine learning in production.

The main focus is not only on building a model, but on understanding how an ML system should be designed, deployed, evaluated, and updated after release. Topics include deployment patterns, monitoring strategies, error analysis, baseline development, concept drift, data quality, label consistency, and project scoping.

In many ways, this course works as an introduction to the rest of the specialization. Later courses go deeper into data pipelines, modeling pipelines, deployment infrastructure, monitoring, and continuous delivery.

## Main Topics Covered

- ML project lifecycle
- Production ML system design
- Deployment patterns
- Model and pipeline monitoring
- Concept drift
- Error analysis
- Model baselines
- Skewed datasets and class imbalance
- Data-centric AI development
- Data augmentation
- Experiment tracking
- Data definition
- Label ambiguity and label consistency
- Human-level performance
- Data pipelines
- Metadata, provenance, and lineage
- Train/dev/test split design
- Project scoping and feasibility analysis

## Course Structure

| Week | Topic | Summary |
|---|---|---|
| Week 1 | Overview of the ML Lifecycle and Deployment | Introduced the main stages of a production ML project and the challenges that appear after a model leaves the development environment. The week covered deployment patterns, monitoring, pipeline monitoring, and practical considerations for deploying deep learning models, including an optional lab with Docker and cloud deployment. |
| Week 2 | Modeling Challenges and Strategies | Focused on common modeling problems in production settings. The week covered why average error is not always enough, how to establish a meaningful baseline, how to perform error analysis, and how to work with skewed datasets, class imbalance, data augmentation, feature additions, and experiment tracking. |
| Week 3 | Data Definition and Baseline | Covered the data side of production ML, including data definition, label ambiguity, label consistency, human-level performance, metadata, data provenance, lineage, balanced data splits, and project scoping. This week also included the final end-to-end project and labs related to data labeling and the ML project lifecycle. |
