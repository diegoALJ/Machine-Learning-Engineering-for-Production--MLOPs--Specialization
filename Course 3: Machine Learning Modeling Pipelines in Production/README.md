# Course 3: Machine Learning Modeling Pipelines in Production

This folder contains the notebooks, Python scripts, labs, and learning materials completed as part of **Course 3: Machine Learning Modeling Pipelines in Production**, from the **Machine Learning for Production (MLOps) Specialization**, offered by **DeepLearning.AI** through **Coursera**.

This course focuses on how to build and optimize machine learning models for production environments. It goes beyond model training and introduces practical techniques for choosing model architectures, managing computational resources, improving training efficiency, analyzing model behavior, and making predictions more interpretable.

The course also covers important production concerns such as model fairness, robustness, explainability, and performance monitoring.

## Course Overview

This course explores the modeling stage of the production ML lifecycle in more depth.

The main idea is that production models need to be accurate, but also efficient, scalable, explainable, and suitable for the environment where they will be served. A model deployed on a cloud server, a mobile device, or an IoT system may have very different requirements in terms of latency, memory, compute, and interpretability.

The course covers methods such as hyperparameter tuning, neural architecture search, dimensionality reduction, quantization, pruning, distributed training, knowledge distillation, model analysis, fairness evaluation, and explainable AI techniques.

## Course Structure

| Week | Topic | Summary |
|---|---|---|
| Week 1 | Neural Architecture Search | Introduced hyperparameter tuning, AutoML, search spaces, search strategies, and neural architecture search. The week also included hands-on work with Keras Tuner, TFX-based model training, and cloud-based AutoML workflows. |
| Week 2 | Model Resource Management Techniques | Focused on reducing model complexity and managing compute, storage, and I/O requirements. The week covered dimensionality reduction, PCA, quantization, quantization-aware training, pruning, and optimization techniques for constrained environments such as mobile and IoT. |
| Week 3 | High-Performance Modeling | Covered techniques for training models more efficiently using distributed processing and parallelism. The week introduced high-performance data ingestion, distributed TensorFlow strategies, training large models, and knowledge distillation with teacher-student networks. |
| Week 4 | Model Analysis | Focused on analyzing, debugging, and improving model behavior after training. The week covered TensorFlow Model Analysis, TFX Evaluator, sensitivity analysis, adversarial attacks, residual analysis, fairness indicators, model remediation, and continuous evaluation. |
| Week 5 | Interpretability | Introduced methods for explaining model predictions to technical and non-technical audiences. The week covered explainable AI, intrinsically interpretable models, model-agnostic methods, partial dependence plots, permutation feature importance, Shapley values, SHAP, LIME, TCAV, and AI Explanations. |
